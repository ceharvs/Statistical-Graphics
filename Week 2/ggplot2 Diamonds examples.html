1. Introductory comments

Leland Wilkinson, author of “The Grammar of Graphics,” created a coherent way to specify (and construct) as wide range of graphics.

When Hadley Wickam was a statistics PhD student at Iowa State he developed a layered grammar based on Lee's work. Hadley implemented this and made it available in R via the ggplot2 package. While he has not yet provided full implementation, what is available will produce a wide range of graphics with relative ease. Hadley is now working for R-Tools Technology Inc. whose product include R-Studio. I heard that one his projects is working on a faster version of ggplot2.

Historically this class made more use of the lattice package. This is a powerful and sophisticated tool for producing a wide range of graphics. ggplot2 also produce a wide range of graphics. The syntax seems to have shorter learning curve and the package is rapidly increasing in popularity.
Hence this class will emphasize ggplot2 more than lattice.

This assignment features part of ggplot qplot because it conveys some of the clarity and power of grammar of graphics specification. While I really like the default white grid lines, gray background and horizontal grid line labels, there are several graphics below where I would change the defaults. Please to not take this assignmentas a complete endorsement of the appearance.
Also note that qplot expects the data to be in a data.frame.

Many of my graphic designs used detailed control of R's base level graphics to address my design tasks. Over time I am revising these the plot production is easier. For example there is now a micromapST package for producing state linked micromaps. Others are also at work making it easier to produce graphics that follow some my designs.

3. Some initial graphics

The plotted x and y variables can be calculated from data.frame variables

qplot(log(carat), log(price), data = diamondSamp)

x*y*z is a new variable related to the volume(see picture page 10)

qplot(carat, x * y * z, data = diamondSamp)

The plot below plots the full data set and takes a while:

qplot(carat, x * y * z, data = diamonds)

There are outliers that limit resolution for the body of the data. Sampling can miss outliers.

4. Categorical variables and geom appearance

A geom is a geometric object such as point, line, polygon, confidence bounds for a regression line and so on.
The default geom is a point.

Lee Wilkinson used the word aesthetics to characterize non-positional aspects of object appearance. For points, the aesthetics are primarily color, size and shape. The qplot() argument key words are colour(or color), size and shape. (Colour is British spelling for the key word.) These are followed respectively by categorical variables encoded as factors. Constants can be provided as argument values using the I() function. The color transparency can be controlled use the alpha keyword.

Note the automatic generation of legends. This really helps in producing decent graphics. 

qplot(carat, price, colour = color, data = diamondSamp)


qplot(carat, price, shape = cut, data = diamondSamp)


qplot(carat, price, color = color, shape = cut, data = diamondSamp)


qplot(carat, price, color = color, size = I(5), data = diamondSamp)


qplot(carat, price, color = color, size = I(5), alpha = I(3/10), data = diamondSamp)

6. Specifying geoms

Geoms are different kinds of geometric objects to render.

  • geom = “point” plots points and is the default geom
  • geom = “line” and “path” plot lines and paths. The line travels from left to right. A path can move in all directions and cross over itself
qplot(displ, hwy, data = mpg, colour = factor(cyl), geom = c("line"), size = I(2))

  • geom = “smooth” produces a smooth approximation to a functional relationship and its standard error.
qplot(carat, price, data = diamondSamp, geom = c("point", "smooth"))

## geom_smooth: method="auto" and size of largest group is <1000, so using loess. Use 'method = x' to change the smoothing method.


qplot(carat, price, data = diamondSamp, geom = c("point", "smooth"), span = 0.2)

## geom_smooth: method="auto" and size of largest group is <1000, so using loess. Use 'method = x' to change the smoothing method.


qplot(carat, price, data = diamondSamp, geom = c("point", "smooth"), span = 1)

## geom_smooth: method="auto" and size of largest group is <1000, so using loess. Use 'method = x' to change the smoothing method.

For modest (n <= 1000) sample sizes ggplot uses loess to smooth. More will be said later about additional packages and smoothes for larger data sets.
